rest api login - entrega final

repositorio:
https://github.com/JoseHVT/Web-practicas_dev.git

requisitos:
1. node.js y npm instalados.
2. mongodb ejecutandose en localhost:27017, o variable MONGODB_URI configurada.

ejecucion:

backend:
cd backend
npm install
npm run dev

frontend:
cd frontend
npm install
npm run dev

urls:
backend: http://localhost:3000
frontend: http://localhost:5173
uso normal: http://localhost:5173/
prueba visual admin: http://localhost:5173/test-api

variables principales:

backend:
- MONGODB_URI
- JWT_SECRET
- JWT_EXPIRES_IN
- PASSWORD_PEPPER
- ALLOWED_ORIGIN

frontend:
- VITE_API_URL

cuentas admin de apoyo:

local:
usuario: tequi
email: t@gmail.com
contrasena: 123456

despliegue:
usuario: root
email: root@mail.com
contrasena: root
rol: admin

para crear o reponer la cuenta demo:
cd backend
npm run seed-demo-admin

para crear o reponer la cuenta del entregable de despliegue:
cd backend
npm run seed-root-admin

endpoints publicos:

- POST /login
- POST /login/register
- GET /
- GET /ping
- GET /marco

endpoints protegidos para cualquier usuario autenticado:

- GET /users
- GET /users/:id
- PUT /login/password
- GET /dashboard/kpis
- GET /dashboard/charts/registrations?startDate=2026-01-01&endDate=2026-12-31

endpoints solo para administradores:

- POST /users
- PUT /users/:id
- DELETE /users/:id
- GET /login
- DELETE /login/:id

seguridad de contrasenas:

el backend usa crypto.scrypt de node.js.
no usa bcrypt ni otra libreria externa para hashing.
las contrasenas se guardan con sal aleatoria y pimienta.
la pimienta se puede configurar con PASSWORD_PEPPER.

prueba de hash:

cd backend
npm run hash-demo

jwt:

el backend usa jsonwebtoken.
el login genera un token firmado.
las rutas protegidas validan el token con el middleware authenticateToken.
las rutas administrativas validan el rol con requireAdmin.

header requerido:
Authorization: Bearer <token>

separacion entre sistema y pruebas:

- `/` y `/dashboard` son el uso normal del sistema.
- `/usuarios` permite consulta a usuarios autenticados y crud completo solo para admins.
- `/test-api` es un banco de pruebas tecnico solo para admins.
