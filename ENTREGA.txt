REST API LOGIN - ENTREGA FINAL

REPOSITORIO:
https://github.com/JoseHVT/Web-practicas_dev.git

REQUISITOS:
1. Node.js y npm instalados.
2. MongoDB ejecutandose en localhost:27017, o variable MONGODB_URI configurada.

EJECUCION:

Backend:
cd backend
npm install
npm run dev

Frontend:
cd frontend
npm install
npm run dev

URLS:
Backend: http://localhost:3000
Frontend: http://localhost:5173
Prueba visual: http://localhost:5173/test-api

ENDPOINTS:

Base:
GET /
GET /ping
GET /marco

Usuarios:
GET /users
GET /users/:id
POST /users
PUT /users/:id
DELETE /users/:id

Login:
POST /login
POST /login/register
GET /login
PUT /login/password
DELETE /login/:id

Dashboard:
GET /dashboard/kpis
GET /dashboard/charts/registrations?startDate=2026-01-01&endDate=2026-12-31

SEGURIDAD DE CONTRASENAS:
El backend usa crypto.scrypt de Node.js.
No usa bcrypt ni otra libreria externa para hashing.
Las contrasenas se guardan con sal aleatoria y pimienta.
La pimienta se puede configurar con PASSWORD_PEPPER.

PRUEBA DE HASH:
cd backend
npm run hash-demo

ACTUALIZAR CONTRASENAS EXISTENTES:
$env:PASSWORD_UPDATES='{"admin":"admin123","user":"user123"}'
npm run update-password-hashes

VALIDACION REALIZADA:
Backend: npm run hash-demo
Frontend: npm run lint
Frontend: npm run build

NOTAS:
El ZIP de entrega no debe incluir carpetas node_modules.
